# 临时核价页线框图 V1

适用场景：
- 临时查询一批商品当前真实成本
- 本次给客户临时报价
- 发现成本错误时，发起正式修正
- 未匹配或低置信匹配时，人工绑定商品并写入对应关系

页面目标：
- 明确区分 `系统真实成本`、`默认报价`、`手动报价`
- 明确展示每条价格和匹配的来源
- 支持不落方案库的临时核价
- 支持在同页内完成“人工确认对应关系”“发起成本修正”

---

## 一、页面结构

一级入口建议：
- 商品与成本
  - 商品列表
  - 临时核价
  - 成本导入记录
  - 异常数据

临时核价路由建议：
- `/catalog/pricing`

## 三、每行展示口径

每条结果建议显示：
- 原始输入
- 匹配商品标准名
- 匹配来源
  - 历史映射
  - 手工确认
  - 标准名精确命中
  - 别名命中
  - 模糊候选
  - 未命中
- 真实成本
  - 显示 `¥金额`
  - 可附一行小字：`成本生效日 2026-03-01`
- 报价
  - 默认按页面顶部点数自动生成
  - 公式：`真实成本 × (1 + 点数/100)`
  - 可手动输入覆盖
  - 手动输入优先级高于自动报价
- 数量
- 成本小计
- 报价小计
- 状态
  - 已核价
  - 待人工确认
  - 缺少真实成本
- 已手动报价
  - 已正式修正

## 四、操作设计

### 1. 确认当前匹配

适用：
- 系统已有候选
- 用户认为当前候选正确

操作后：
- 写入对应关系表
- 标记来源为 `人工确认`
- 当前行立即刷新
- 总成本立即重算

按钮文案：
- `确认当前匹配`

---

### 2. 绑定商品

适用：
- 当前未匹配
- 当前候选不对

交互：
- 行内点击 `绑定商品`
- 右侧弹层打开

```text
+-----------------------------------------------------------+
| 绑定商品                                                  |
| 原始输入：韩国东大门时尚百搭发圈10cm                      |
| 标准化后：韩国东大门时尚百搭发圈10cm                      |
|-----------------------------------------------------------|
| 搜索标准商品                                               |
| [ 发圈                                                   ] |
|-----------------------------------------------------------|
| 候选结果                                                   |
| ( ) 发圈              分类：饰品      当前成本：¥0.80      |
| ( ) 韩版大号发圈      分类：饰品      当前成本：¥1.10      |
|-----------------------------------------------------------|
| [取消]                                      [确认绑定]     |
+-----------------------------------------------------------+
```

确认后：
- 写入对应关系表
- 当前行立即刷新
- 总成本立即重算
- 若当前行没有手动报价，报价按当前页面点数自动生成

---

### 3. 成本有误

适用：
- 当前显示的系统真实成本不对

交互：
- 行内点击 `成本有误`
- 弹出双动作窗口

```text
+------------------------------------------------------------------------------------------------------+
| 修正成本                                                                                             |
| 商品：蜜斯莉去屑清爽山茶花洗发水248g                                                                 |
| 当前真实成本：¥6.50                                                                                  |
| 成本生效日：2026-03-01                                                                               |
|------------------------------------------------------------------------------------------------------|
| 最近成本历史                                                                                         |
| 2026-03-01  ¥6.50                                                                                    |
| 2026-02-01  ¥6.20                                                                                    |
| 2025-12-15  ¥6.80                                                                                    |
|------------------------------------------------------------------------------------------------------|
| 本页报价规则                                                                                         |
| 默认报价点数：25%                                                                                    |
| 默认报价：系统将按“新真实成本 × 1.25”自动重算                                                       |
| 如果当前行已有手动报价，则保留手动报价不变                                                          |
|------------------------------------------------------------------------------------------------------|
| 新成本          [ 6.10 ]                                                                             |
| 生效时间        [ 2026-03-23 ]                                                                       |
| 来源            [ 手工录入 v ]                                                                        |
| 供应商          [ 供应商A ]                                                                          |
| 修正原因        [ 本次采购口头确认价 ]                                                                |
|------------------------------------------------------------------------------------------------------|
| [取消]                                                            [保存并重算]                      |
+------------------------------------------------------------------------------------------------------+
```

规则：
- 该入口不再提供“仅本次临时调整”
- 该入口只用于更新正式成本
- 保存时新增一条 `product_cost_history`
- 不覆盖历史
- 保存后当前结果立即重算
- 若某行报价来自自动点数，则按新的真实成本重新计算报价
- 若某行报价是手动填写，则保持手动报价不变

## 五、顶部总览联动规则

总览区应始终固定显示：
- 总成本
- 总报价
- 总毛利
- 毛利率
- 待人工确认
- 缺少真实成本
- 手动报价项数
- 当前报价点数

联动规则：
- 确认匹配后，立即重算总成本
- 绑定商品后，立即重算总成本
- 更新正式成本后，立即重算总成本和毛利
- 修改页面点数后，所有未手动覆盖的报价立即重算
- 更新客户报价后，立即重算总报价和毛利

说明文案建议：
- `默认报价按当前点数自动生成，手动填写报价优先级更高`
- `本页不保存方案`

## 六、来源标识建议

价格来源：
- 真实成本
- 自动报价
- 手动报价
- 无真实成本

匹配来源：
- 历史映射
- 手工确认
- 精确匹配
- 别名匹配
- 模糊候选
- 未匹配

映射处理结果：
- 已自动写入对应表
- 已人工确认写入
- 待人工确认
- 未写入

## 八、推荐实现顺序

第一阶段：
- 补“绑定商品”弹层
- 补“确认当前匹配”后总价实时重算
- 补全局报价点数输入和自动报价联动
- 补来源标签显示

第二阶段：
- 补“成本有误”弹窗
- 补手动报价覆盖规则
- 加总报价、毛利、毛利率

第三阶段：
- 导出本次临时核价结果
- 支持保存为临时报价单
- 支持历史查询

---

## 九、结论

临时核价页最重要的不是把“价格”显示出来，而是把三层东西拆清楚：
- 系统真实成本
- 按点数自动生成的默认报价
- 手动覆盖后的最终报价

同时要保证两个闭环：
- 未匹配/待确认商品可以在本页内完成绑定并写入对应表
- 发现成本错误时，可以在本页内发起修正，但正式成本必须通过新增历史记录的方式落库
